# KMP 技术文章开源项目入口优化设计

## 背景

KMP 三端性能技术文章当前使用卡片展示相关开源项目。卡片标题可以点击，但页面没有直接显示链接地址，也没有明确的操作文案；提交哈希与“公开仓库”的含义同样缺少解释，读者容易把这一节误解为性能 Demo 下载入口。

## 目标

- 明确说明本节展示的是文章涉及的开源项目及测试版本，不是性能 Demo 下载入口。
- 让读者无需试探即可识别并打开源码仓库。
- 区分“本次测试锁定提交”与“对照方案源码”两类信息。
- 保持现有文章整体视觉风格、深浅色模式和移动端适配不变。

## 页面设计

章节标题由“公开源码入口”调整为“相关开源项目与测试版本”。章节说明补充以下语义：

1. 项目名称、完整 URL 和操作入口均可打开对应公开仓库。
2. CPF-KMP Kotlin 与 AKInterop 的长字符串是本次验证锁定的 Git 提交哈希。
3. KuiklyBase 与 KNOI 是对照方案的公开源码。
4. 这些链接不是三端性能 Demo 的下载入口。

每张项目卡片按固定顺序展示：

1. 项目名称。
2. 项目在文章中的用途说明。
3. 完整、可点击的源码 URL。
4. 版本类型及值：`本次测试锁定提交` 或 `对照方案源码`。
5. 明确操作文案：`打开源码仓库 →`。

所有外部链接在新标签页打开，并保留安全的 `rel` 属性。完整 URL 在窄屏允许换行，不能造成横向滚动。

## 数据结构

开源项目数据继续集中维护在 `kmpThreePlatformPerformance.mjs`，每项增加：

- `description`：项目用途。
- `revisionLabel`：版本字段标签。
- `actionLabel`：操作文案。

项目名称、URL 和版本值仍来自同一数据项，组件不重复硬编码链接。

## 测试与验收

自动化测试覆盖：

- 新章节标题和解释文案存在。
- 四个仓库的完整 URL 均以可见文本展示，并与链接目标一致。
- 四张卡片均显示“打开源码仓库 →”。
- CPF-KMP Kotlin 与 AKInterop 显示“本次测试锁定提交”。
- KuiklyBase 与 KNOI 显示“对照方案源码”。
- 桌面端和移动端均无横向溢出。
- 全站构建、已有 KMP 数据测试及其他文章测试继续通过。

## 不在本次范围

- 不公开或上传三端性能 Demo 源码。
- 不修改文章性能数据和技术选型结论。
- 不调整站点导航、文章 URL 或其他文章页面。
